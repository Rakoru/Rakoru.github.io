ISO/OSI Model and Network Protocols

A comprehensive guide to the OSI model layers and protocol assignments

ISO/OSI Model Description

The Open Systems Interconnection (OSI) model is a conceptual framework standardized by the International Organization for Standardization (ISO). It describes how different network protocols interact and work together to provide network services. The model divides network communication into seven abstraction layers, each serving specific functions and communicating with the layers directly above and below it.

This layered approach allows for modular network design, where protocols at each layer can be developed and updated independently without affecting the entire system. The OSI model serves as a reference for understanding network architecture, troubleshooting network issues, and designing new network protocols.

OSI Layers, Protocol Data Units, and Protocol Assignments

Layer 7 - Application Layer

PDU: Data
Protocols:
  • DHCP - Dynamic Host Configuration Protocol
  • DNS - Domain Name System
  • FTP - File Transfer Protocol
  • HTTP - Hypertext Transfer Protocol
  • MIME - Multipurpose Internet Mail Extensions
  • NetBIOS - Network Basic Input/Output System
  • NNTP - Network News Transfer Protocol
  • SIP - Session Initiation Protocol
  • Telnet - Teletype Network

Layer 6 - Presentation Layer

PDU: Data
Protocols:
  • SSL - Secure Sockets Layer
  • TLS - Transport Layer Security

Layer 5 - Session Layer

PDU: Data
Protocols:
  • NetBIOS - Network Basic Input/Output System (Session)
  • RTP - Real-time Transport Protocol

Layer 4 - Transport Layer

PDU: Segment (TCP) / Datagram (UDP)
Protocols:
  • TCP - Transmission Control Protocol
  • UDP - User Datagram Protocol

Layer 3 - Network Layer

PDU: Packet
Protocols:
  • ICMP - Internet Control Message Protocol
  • IGMP - Internet Group Management Protocol
  • IPv4 - Internet Protocol version 4
  • IPv6 - Internet Protocol version 6

Layer 2 - Data Link Layer

PDU: Frame
Protocols:
  • ARP - Address Resolution Protocol
  • IEEE 802.11 - Wireless LAN (Wi-Fi)
  • L2TP - Layer 2 Tunneling Protocol
  • SLIP - Serial Line Internet Protocol

Layer 1 - Physical Layer

PDU: Bit
Protocols/Standards:
  • Bluetooth - Wireless technology standard
  • IEEE 1394 - FireWire
  • RS-232 - Serial communication standard
  • RS-449 - Serial communication standard
  • USB - Universal Serial Bus

Network Technologies

Physical Layer Technologies

Photo 1

USB (Universal Serial Bus)

USB is an industry standard for cables, connectors, and protocols for connection, communication, and power supply between computers and peripheral devices. It provides both data transfer and power delivery capabilities.

Photo 2

Bluetooth

Bluetooth is a short-range wireless technology standard used for exchanging data between fixed and mobile devices over short distances using UHF radio waves. It's commonly used for wireless peripherals and IoT devices.

Data Link Layer Protocols

Photo 3

IEEE 802.11 (Wi-Fi)

IEEE 802.11 is a set of protocols that implement wireless local area network (WLAN) communication. It defines the physical and data link layers for wireless networking and is commonly known as Wi-Fi.

Photo 4

ARP (Address Resolution Protocol)

ARP is a protocol used to map IP network addresses to the hardware (MAC) addresses used by a data link protocol. It's essential for local network communication in IPv4 networks.

Network Layer Protocols

Internet Protocol Suite

Photo 5

IPv4 (Internet Protocol version 4)

IPv4 is the fourth version of the Internet Protocol and is one of the core protocols of standards-based internetworking methods. It uses 32-bit addresses and is the most widely deployed protocol.

Photo 6

IPv6 (Internet Protocol version 6)

IPv6 is the most recent version of the Internet Protocol, designed to eventually replace IPv4. It uses 128-bit addresses, providing a vastly larger address space and improved features for routing and network autoconfiguration.

Supporting Network Protocols

Photo 7

ICMP (Internet Control Message Protocol)

ICMP is used by network devices to send error messages and operational information. It's commonly used for diagnostic purposes, such as the ping command to test connectivity.

Photo 8

IGMP (Internet Group Management Protocol)

IGMP is used by hosts and adjacent routers to establish multicast group memberships. It's an essential protocol for IP multicast communications.

Transport Layer Protocols

Connection-Oriented Protocol

Photo 9

TCP (Transmission Control Protocol)

TCP is a connection-oriented protocol that provides reliable, ordered, and error-checked delivery of data between applications. It's used when reliability is critical, such as in web browsing and file transfers.

Photo 10

UDP (User Datagram Protocol)

UDP is a connectionless protocol that provides a simple, fast method of sending datagrams. It's used when speed is more important than reliability, such as in streaming media and online gaming.

Real-time Communication

Photo 11

RTP (Real-time Transport Protocol)

RTP is designed for delivering audio and video over IP networks. It's used in streaming media systems, video conferencing, and push-to-talk systems.

Photo 12

SIP (Session Initiation Protocol)

SIP is a signaling protocol used for initiating, maintaining, and terminating real-time sessions that include voice, video, and messaging applications. It's widely used in VoIP communications.

Application Layer Protocols

Web and File Transfer

Photo 13

HTTP (Hypertext Transfer Protocol)

HTTP is the foundation of data communication for the World Wide Web. It defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.

Photo 14

FTP (File Transfer Protocol)

FTP is a standard network protocol used for the transfer of files between a client and server on a computer network. It's built on a client-server model architecture using separate control and data connections.

Network Services

Photo 15

DNS (Domain Name System)

DNS is a hierarchical distributed naming system for computers, services, or other resources connected to the Internet or a private network. It translates domain names to IP addresses.

Photo 16

DHCP (Dynamic Host Configuration Protocol)

DHCP is a network management protocol used to dynamically assign IP addresses and other network configuration parameters to devices on a network, enabling them to communicate on IP networks.
