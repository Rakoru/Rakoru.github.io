DNS and Mail System Report

Part 1: Theoretical

1.1 DNS System

1.1.1 Top Level Domains (TLDs)

Top Level Domains are the last part of a domain name, for example: .com, .org, .net, .edu, .gov, .uk.

1.1.2 DNS Hierarchy

DNS has a hierarchical structure: root servers, top level domains, and domain names. DNS queries ask for an IP address. DNS delegation passes control of a domain to another DNS server.

1.1.3 DNS Records and Reverse DNS

DNS records store information like IP addresses (A record). Reverse DNS finds a domain name from an IP address.

1.1.4 Authoritative vs Non‑Authoritative Answer

An authoritative answer comes from the main DNS server for the domain. A non‑authoritative answer comes from cache.

1.1.5 DNS Resolver and Cache

A DNS resolver looks up domain names. DNS cache stores results to speed up future requests. The file /etc/hosts can map names to IPs locally.

1.2 Mail System

1.2.1 Mail Protocols

SMTP sends e‑mail. POP3 and IMAP receive e‑mail. Communication follows a client‑server model using basic text commands.

1.2.2 NNTP Protocol

NNTP is used for reading and posting messages to newsgroups.

1.2.3 Mailbox and Address

An e‑mail mailbox stores messages. An e‑mail address has a user name and domain name (user@example.com).

1.2.4 E‑mail Structure

An e‑mail has a header and body. A blank line separates them.

1.2.5 Attachments and MIME

Attachments are files sent with e‑mail. MIME allows different file types and text encoding.

1.2.6 Mailing Lists and Mail Gateways

Mailing lists send messages to many users. Mail gateways connect different mail systems.

1.2.7 E‑mail Security

SSL and TLS encrypt e‑mail traffic. PGP provides encryption and digital signatures.

1.2.8 Spam Protection

Spam is unwanted e‑mail. Blacklists, graylists, and spam filters help block spam.
