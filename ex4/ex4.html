TCP and UDP Report

Part 1: Theoretical

1.1 TCP Protocol

TCP (Transmission Control Protocol) is a reliable protocol. It makes sure data arrives in the correct order and without errors.

1.2 UDP Protocol

UDP (User Datagram Protocol) is a fast but unreliable protocol. It sends data without checking delivery.

1.3 TCP Protocol Parameters

Important TCP parameters include source port, destination port, sequence number, acknowledgment number, and flags.

1.4 UDP Protocol Parameters

UDP parameters are source port, destination port, length, and checksum.

1.5 TCP Connection

A TCP connection is a stable communication link between two devices.

1.6 Three-Way Handshake

The three-way handshake is used to start a TCP connection: SYN, SYN-ACK, and ACK.

1.7 Connection Establishment and Release

Connection establishment starts communication. Connection release safely closes the TCP connection.

1.8 Ports

Ports are numbers that identify applications on a device (example: HTTP uses port 80).
